# Dota Spy

Веб-игра в стиле "Шпион" на тематику Dota 2.

## Структура проекта

Проект содержит две версии игры:

### 1. Многопользовательская версия (рекомендуемая)
Расположена в папке `public/`:
- `public/index.html` - главная страница
- `public/lobby.html` - лобби игры
- `public/game.html` - игровая комната
- `public/style.css` - стили
- `public/heroes.js` - список героев Dota 2

### 2. Одиночная версия
Расположена в корне:
- `index.html` - настройка одиночной игры
- `game.html` - игровая комната
- `style.css` - стили
- `heroes.js`, `locations.js` - данные

## Запуск

### Многопользовательская версия:
```bash
node server.js
```
Сервер запустится на порту 3000: http://localhost:3000

### Требования:
- Node.js
- Зависимости: `npm install`

## Как играть

1. **Создатель комнаты**: Открывает главную страницу, создает комнату, получает код
2. **Другие игроки**: Заходят на тот же сайт, вводят код комнаты
3. **Лобби**: Хост настраивает игру (количество игроков, шпионов, время)
4. **Игра**: После начала всем показывается случайный герой, кроме шпиона
5. **Голосование**: Игроки могут голосовать за изгнание подозреваемого
6. **Победа**: Шпион выигрывает, если не пойман; мирные выигрывают, если шпион изгнан

## 🚀 Деплой на хостинг

### Опции хостинга:

#### 1. **Railway** (рекомендуется)
```bash
# Установите Railway CLI
npm i -g @railway/cli

# Логин
railway login

# Инициализируйте проект
railway init

# Деплой
railway up
```

#### 2. **Render**
1. Зарегистрируйтесь на https://render.com
2. Создайте новый Web Service
3. Подключите ваш репозиторий
4. Используйте `render.yaml` для автоматической настройки

#### 3. **Vercel** (для фронтенда + отдельный сервер)
```bash
# Установите Vercel CLI
npm i -g vercel

# Деплой
vercel
```

#### 4. **Локальный хостинг с ngrok** (для тестирования)
```bash
# Установите ngrok
# Запустите сервер
node server.js

# В другом терминале
ngrok http 3000
```

## 🔧 Переменные окружения

## 🛠️ Отладка

Если возникли проблемы:
1. Проверьте логи сервера
2. Убедитесь, что порт открыт для входящих соединений
3. Проверьте поддержку WebSocket на хостинге
4. Для HTTPS используйте `wss://` вместо `ws://`

## 📞 Поддержка

Если что-то не работает:
- Проверьте консоль браузера (F12 → Console)
- Проверьте логи сервера на хостинге
- Убедитесь, что все игроки используют одну версию сайта
